The Making of Antagonista

Static Site Generator

Since I want to do it like a real front-end project, I prefer a pure javescript workflow to something ruby-based like Jekyll. With some googling I found this summarization, and by looking around the candidates I decided to give wintersmith a try.

Wintersmith is ok, but it is not quite grunt aware, sometime I had to reimplement some grunt-tasks like compressing and concatenating. After some frustrations I began to find another solution.

Then I found the awesome assemble, which is exactly built for grunt. It is highly customizable and only focusing on generating html however you want, leaving other js and css tasks back to other grunt. Perfect!

Markdown

Another thing I want is being able to write using markdown, who dosen't? Assemble supports markdown by utilizing marked, nothing much to worry about.

One thing I like about octopress is the blogging rake tasks. Still not a big deal with grunt, actually it is just a grunt new-post I have to implement.

Deployment

Again I got much inspiration from octopress' branching. I'll just have a source branch for source code and commit final files to master branch for serving.

There are two tasks I have to implement:

  • grunt setup: setup the dist/ folder for final stuffs in master branch.
  • grunt deploy: push updates to master
